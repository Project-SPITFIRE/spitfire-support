The Configuration Centre provides administrators with comprehensive control over system roles, permissions, and access management. Create custom roles, manage existing permissions, and configure system-wide access controls.

## Who can access Configuration Centre

<Badge text="Admin" variant="danger" /> Full access to all configuration features and role management

<Badge text="Ops Support" variant="caution" /> Limited access to view configurations and assist with role-related issues

<Badge text="All Others" variant="note" /> No access to Configuration Centre features

## What is Configuration Centre?

The Configuration Centre is the central hub for:
- **Role Management** - Creating and modifying system roles
- **Permission Control** - Defining what each role can access
- **Access Configuration** - Setting up page and feature-level permissions
- **System Security** - Managing overall access controls and restrictions

## Best Practices

:::tip[Configuration Centre Best Practices]
- **Document Role Purposes:** Clearly define what each role is intended for
- **Regular Permission Reviews:** Audit roles quarterly to ensure they remain appropriate
- **Test New Roles:** Verify permissions work as expected before assigning to users
- **Minimal Permissions:** Start with basic permissions and add more as needed
- **Backup Configurations:** Keep records of role configurations for reference
:::

:::caution[Security Considerations]
- Never grant more permissions than necessary for a role
- Regularly review high-privilege roles like Admin and Ops Support
- Monitor permission changes through system logs
- Ensure role changes are properly documented and approved
:::

## Troubleshooting

<Accordion>
  <AccordionItem title="User Cannot Access Expected Features" icon="tabler:alert-triangle">
    <p>Check role permissions and assignments.</p>
    <ul>
      <li>Verify user's role assignment in Account Management</li>
      <li>Check role permissions in Configuration Centre</li>
      <li>Confirm required permissions are enabled for the role</li>
      <li>Test permissions with a test account if needed</li>
    </ul>
  </AccordionItem>

  <AccordionItem title="Role Permissions Not Working" icon="tabler:bug">
    <p>Troubleshoot permission configuration issues.</p>
    <ul>
      <li>Verify permission changes have been saved</li>
      <li>Check for permission dependencies and prerequisites</li>
      <li>Ensure user has logged out and back in after changes</li>
      <li>Review system logs for permission-related errors</li>
    </ul>
  </AccordionItem>

  <AccordionItem title="Cannot Modify Existing Roles" icon="tabler:lock">
    <p>Resolve role editing and modification issues.</p>
    <ul>
      <li>Confirm you have Admin access to Configuration Centre</li>
      <li>Check if role is currently in use by active users</li>
      <li>Verify no system-level restrictions on the role</li>
      <li>Ensure proper admin permissions for role modification</li>
    </ul>
  </AccordionItem>
</Accordion>
