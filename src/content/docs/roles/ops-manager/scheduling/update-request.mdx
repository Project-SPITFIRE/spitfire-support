You can modify requests you've submitted to reflect operational changes and new information.

## Accessing your requests

1. Navigate to **Scheduling** â†’ **Submit Request**
2. Find your request in the **Submit Requests** table
3. Click the request row to open **Request Details**

## How to update a request

1. Locate your request in the table
2. Click the request row to open details
3. Modify fields as needed
4. Click **Save** or **Update**

## When to update requests

- **Requirements change**: Operational needs evolve
- **Timeline adjustments**: Dates shift based on schedule changes
- **Missing information**: Ops Support requests clarification
- **Resource modifications**: Manager count changes

## Responding to information requests

When status shows <Badge text="Missing Information" variant="danger" />:

1. Open request details and review Ops Support notes
2. Update relevant fields with requested information
3. Add clarifying notes in description
4. Save changes to update status

## Best practices

- **Update promptly**: Make changes as soon as you're aware
- **Clear communication**: Explain what changed and why
- **Coordinate**: Inform Ops Support of significant changes
- **Document changes**: Keep detailed modification notes

:::note[Major changes to in-progress requests]
For significant changes to requests already being worked on, contact Ops Support directly. Major scope changes may require a new request.
:::

## After updating

1. Changes save immediately
2. Ops Support is notified
3. Request may be re-reviewed
4. Status may update based on new information
5. Timeline may be adjusted accordingly
